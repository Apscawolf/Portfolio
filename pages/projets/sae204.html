SAE 2.04

Exploitation d'une base de données

Conception et exploitation d'une base de données relationnelle sur le thème du ski alpin avec PostgreSQL

📅 Mai 2025 👥 Travail en équipe 🗄️ Base de données 💻 PostgreSQL

Aperçu du Projet

Conception complète d'une base de données relationnelle sur le thème du ski alpin, de la modélisation conceptuelle à l'exploitation avec des requêtes complexes.

🎯 Objectif

Concevoir et implémenter une base de données complète pour gérer les informations d'une station de ski : pistes, remontées, forfaits, météo.

👥 Équipe

Projet en binôme avec répartition des tâches : modélisation, création des tables, insertion des données et requêtes d'exploitation.

🏆 Résultats

Base de données fonctionnelle avec 15+ tables, contraintes d'intégrité, triggers et requêtes complexes pour l'exploitation des données.

Modélisation de la Base de Données

Approche méthodologique complète depuis l'analyse des besoins jusqu'au modèle physique.

📋 Analyse des besoins

  • Étude du domaine : Fonctionnement d'une station de ski
  • Identification des entités : Pistes, remontées, forfaits, skieurs
  • Définition des règles métier et contraintes
  • Spécification des besoins d'interrogation

🔧 Modèle conceptuel (MCD)

  • Entités principales : Station, Piste, Remontée, Forfait
  • Relations identifiées : desservir, emprunter, proposer
  • Attributs et cardinalités définis précisément
  • Validation du modèle avec des scénarios d'usage

💾 Modèle physique (MPD)

  • Transformation en tables PostgreSQL
  • Définition des clés primaires et étrangères
  • Contraintes d'intégrité et de domaine
  • Optimisation des index pour les performances

Structure de la Base de Données

Organisation des données en tables relationnelles avec contraintes d'intégrité rigoureuses.

🏔️ Gestion des stations

  • Table Station : Informations générales, localisation
  • Table Domaine : Domaines skiables reliés
  • Table Météo : Conditions quotidiennes
  • Relations : Une station peut avoir plusieurs domaines

🎿 Pistes et remontées

  • Table Piste : Difficulté, longueur, dénivelé
  • Table Remontée : Type, capacité, vitesse
  • Table Liaison : Connexions entre pistes
  • Contraintes : Cohérence des niveaux de difficulté

🎫 Forfaits et tarification

  • Table Forfait : Types, durées, prix
  • Table Tarif : Grille tarifaire saisonnière
  • Table Réduction : Promotions et conditions
  • Historique : Évolution des prix dans le temps

👤 Utilisateurs et fréquentation

  • Table Skieur : Profils et préférences
  • Table Passage : Utilisation des remontées
  • Table Statistique : Données d'affluence
  • Triggers : Mise à jour automatique des compteurs

Requêtes et Exploitation

Développement de requêtes SQL complexes pour répondre aux besoins métier de la station.

📊 Analyses statistiques

  • Fréquentation par piste et par période
  • Chiffre d'affaires par type de forfait
  • Taux d'utilisation des remontées mécaniques
  • Corrélation météo/affluence avec fonctions fenêtre

🔍 Requêtes de gestion

  • Recherche de pistes par niveau et conditions
  • Calcul des temps de trajet entre points d'intérêt
  • Optimisation des parcours selon les préférences
  • Gestion des capacités et files d'attente

📈 Tableaux de bord

  • Vues matérialisées pour les KPI
  • Requêtes de synthèse avec GROUP BY complexes
  • Calculs de rentabilité par secteur
  • Prévisions basées sur l'historique

⚙️ Fonctions avancées

  • Procédures stockées pour les traitements batch
  • Triggers pour la cohérence des données
  • Fonctions personnalisées de calcul
  • Index composites pour l'optimisation

Défis Techniques Surmontés

Les principales difficultés rencontrées et les solutions mises en place.

🔗 Intégrité référentielle

Défi : Maintenir la cohérence entre tables fortement liées (pistes-remontées-domaines).

Solution : Contraintes de clés étrangères en CASCADE et triggers de validation pour les règles métier complexes.

⚡ Performances

Défi : Optimiser les requêtes sur de gros volumes de données de passage.

Solution : Index composites stratégiques, partitionnement par date et requêtes avec EXPLAIN ANALYZE.

📊 Requêtes complexes

Défi : Créer des analyses statistiques avec jointures multiples et fonctions fenêtre.

Solution : Décomposition en vues intermédiaires et utilisation des CTE (Common Table Expressions).

Réflexion et Apprentissages

💡 Ce que j'ai appris

Ce projet m'a fait comprendre l'importance de la phase de modélisation en amont. Une base bien conçue facilite énormément l'écriture des requêtes et la maintenance. J'ai aussi découvert la richesse du langage SQL au-delà des requêtes basiques : les fonctions fenêtre, les CTE et les triggers ouvrent de nombreuses possibilités.

L'aspect performance m'a particulièrement intéressé. Comprendre comment PostgreSQL exécute les requêtes grâce à EXPLAIN ANALYZE et savoir où placer les index fait une énorme différence sur les gros volumes de données.

Le travail en binôme sur un projet technique de cette ampleur nous a appris à nous coordonner efficacement et à valider mutuellement notre travail, ce qui est essentiel en développement de bases de données.
